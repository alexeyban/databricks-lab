     Plan: Propagate Source Data Types Through DV Generator to Satellites

     Context

     The DV 2.0 generator (generators/dv_generator/) already infers column data
     types in
     Step 1 (stored in ColumnDef.data_type and 01_schema_analysis.json), but
     these types
     are dropped when Step 2 builds SatDef objects ‚ÄĒ satellites only carry
     tracked_columns
     as a plain list of strings. As a result:

     - dv_model.json has no structured type info per tracked column (only type
     hints embedded
     in human-readable column_descriptions text)
     - create_sat_table() in NB_dv_metadata.ipynb creates all satellite payload
     columns
     as STRING, regardless of their true source type

     The fix: carry column types all the way from Step 2 ‚Üí SatDef.column_types ‚Üí
      dv_model.json
     ‚Üí create_sat_table() DDL.

     ---
     Changes

     1. generators/dv_generator/models.py

     Add column_types: dict[str, str] field to SatDef (column name ‚Üí canonical
     type):

     @dataclass
     class SatDef:
         ...
         tracked_columns: list[str] = field(default_factory=list)
         column_types: dict[str, str] = field(default_factory=dict)   # NEW
         ...

     2. generators/dv_generator/steps/step2_rule_engine.py

     In _classify_satellites(), build a col_type_map from the TableDef before
     creating
     SatDefs, then pass it to each satellite definition:

     # When constructing SatDef:

         ...

         ...

         ...

     which will
